# Git Workflow

## Схема роботи

```text
┌─────────────────────────────────────────────────────────────────┐
│                    ВИКЛАДАЧ (main repo)                          │
│                  github.com/victorchei/php-labs                  │
│                                                                  │
│   ┌──────────┐  ┌──────────┐  ┌────────────────────────────┐   │
│   │  shared/ │  │  demo/   │  │  variants/v1.md — v30.md   │   │
│   └──────────┘  └──────────┘  └────────────────────────────┘   │
└─────────────────────────┬───────────────────────────────────────┘
                          │
          ┌───────────────┼───────────────┐
          │ fork          │ fork          │ fork
          ▼               ▼               ▼
┌─────────────────┐ ┌─────────────────┐ ┌─────────────────┐
│   Студент 1     │ │   Студент 2     │ │   Студент N     │
│   (працює v1)   │ │   (працює v2)   │ │   (працює vN)   │
└─────────────────┘ └─────────────────┘ └─────────────────┘
```

## Для студентів

### Початкове налаштування (один раз)

1. Відкрийте: <https://github.com/victorchei/php-labs>
2. Натисніть **Fork** (правий верхній кут)
3. Clone свій форк:

```bash
git clone https://github.com/ВАШ_ЛОГІН/php-labs.git
cd php-labs
git remote add upstream https://github.com/victorchei/php-labs.git
```

4. Запустіть сервер:

```bash
php -S localhost:8000
```

### Як працювати над завданням

1. Відкрийте свій варіант: `lr1/variants/vN.md` (де N — ваш номер)
2. Прочитайте завдання — кожен розділ описує що потрібно зробити
3. Подивіться на **demo/** — це робочий приклад виконання (з іншими даними)
4. Створіть файли відповідно до завдання, використовуючи demo як зразок
5. Перевірте у браузері: `http://localhost:8000/lr1/...`

```text
РОБОЧИЙ ПРОЦЕС:

1. Прочитай vN.md (завдання)
2. Подивись demo/ (приклад)
3. Напиши код
4. Перевір у браузері
5. git add . && git commit && git push
```

### Структура проєкту

```text
php-labs/
├── shared/          # Спільні стилі (не змінювати)
├── lr1/
│   ├── demo/        # Робочий приклад (дивитись, не змінювати)
│   ├── variants/
│   │   ├── v1.md    # Завдання для варіанту 1
│   │   ├── v2.md    # Завдання для варіанту 2
│   │   └── ...
│   └── assignment.md # Загальний опис лаби
├── lr2/, lr3/, lr4/  # Аналогічно
```

### Синхронізація з upstream

```bash
git fetch upstream
git checkout main
git merge upstream/main
git push
```

### Уникнення конфліктів

- Працюйте **ТІЛЬКИ** зі своїм варіантом
- Не змінюйте `shared/`, `demo/`, інші варіанти

## Для викладача

### Що оновлюється автоматично

| Що оновлюється              | Студент отримає |
| --------------------------- | --------------- |
| `shared/`                   | Так             |
| `demo/`                     | Так             |
| `docs/`                     | Так             |
| `lr2/`, `lr3/` (нові лаби)  | Так             |
| Нові варіанти               | Так             |

### Перевірка робіт

1. Студент надсилає посилання на свій форк
2. Викладач клонує/переглядає форк
3. Порівнює з demo/ та vN.md (завданням)

## Типові помилки

| Проблема                  | Рішення                                                                         |
| ------------------------- | ------------------------------------------------------------------------------- |
| `permission denied`       | Перевірте SSH ключі або використовуйте HTTPS                                    |
| Конфлікт при merge        | Працюйте тільки зі своїм варіантом                                             |
| Сторінка не працює        | Перевірте синтаксис: `php -l file.php`                                          |
| Сервер не запускається    | Перевірте PHP: `php -v`                                                         |
| `upstream` не знайдено    | `git remote add upstream https://github.com/victorchei/php-labs.git`            |
