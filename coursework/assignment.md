## 2. Технічний стек

Курсову можна виконувати **на одному з двох стеків**:

### Опція A — Laravel (рекомендовано, до 100 балів)

| Компонент     | Вимога                   |
|---------------|--------------------------|
| Мова          | PHP 8.x                  |
| Фреймворк     | Laravel 10/11            |
| БД            | MySQL / MariaDB          |
| ORM           | Eloquent (модель = клас в `app/Models/`) |
| Шаблонізатор  | Blade                    |
| Auth          | Laravel Breeze           |
| Версіонування | Git + GitHub             |

### Опція B — Vanilla PHP (на базі ЛР5, до ~80 балів)

| Компонент     | Вимога                   |
|---------------|--------------------------|
| Мова          | PHP 8.x                  |
| Архітектура   | Власна MVC (з ЛР4 + Model-шар від себе) |
| БД            | MySQL / MariaDB через PDO |
| Доступ до БД  | `classes/Database.php` (PDO singleton) + класи в `models/` |
| Шаблонізатор  | PHP includes + `ob_start` (з ЛР4 `renderDemoLayout`) |
| Auth          | Власна форма + `password_hash` (з ЛР5) |
| Версіонування | Git + GitHub             |

**Деталі вибору + порівняння можливостей** → [feature-catalog.md § 0](feature-catalog.md#0-%D1%81%D1%82%D0%B5%D0%BA-laravel-%D0%B0%D0%B1%D0%BE-vanilla-php-%D1%8F%D0%BA-%D0%BB%D1%805)

### ⛔ MVC — обов'язково на обох стеках

3 фізичні шари: **Model** (доступ до БД), **View** (шаблони), **Controller** (оркестрація). Жодного SQL у контролері/view, жодного echo/HTML у моделі. Кожна з 3+ сутностей — окрема модель. Деталі → [feature-catalog.md § MVC](feature-catalog.md#%E2%9B%94-mvc--%D0%BE%D0%B1%D0%BE%D0%B2%D1%8F%D0%B7%D0%BA%D0%BE%D0%B2%D0%B0-%D0%B2%D0%B8%D0%BC%D0%BE%D0%B3%D0%B0-%D0%B4%D0%BB%D1%8F-%D0%BE%D0%B1%D0%BE%D1%85-%D1%81%D1%82%D0%B5%D0%BA%D1%96%D0%B2)

## 3. Функціональні вимоги

### Обов'язкові модулі

| Модуль     | Деталі                                                          |
|------------|-----------------------------------------------------------------|
| Auth       | Реєстрація, вхід, вихід, ролі (адмін/користувач)                |
| CRUD       | Мінімум 3 пов'язані сутності (з FK)                             |
| Валідація  | Request-класи, зрозумілі повідомлення про помилки               |
| Міграції   | Всі схеми БД через `php artisan make:migration`                 |
| Пагінація  | 10-20 записів на сторінку                                       |
| Пошук      | По одній сутності (за назвою/іменем)                            |
| Зображення | Завантаження + `storage/app/public` + symlink                   |

### Додаткові модулі (опціонально, +бали)

- Email-сповіщення (SMTP / Mailtrap)
- REST API (Sanctum token auth)
- Багатомовність (`resources/lang`)
- Datatables / DataTables.net
- Експорт у PDF / Excel
- Інтеграція з зовнішнім API
